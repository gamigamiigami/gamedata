# Firebase 設定手順（学校システムを有効にする）

学校登録フォーム（`/teachers/register/`）と管理者ダッシュボード（`/admin/`）を
**実際に動かすため**の一度きりの設定です。所要 約10分。Firebase Console での作業です。

> ⚠️ この設定をするまで、登録フォームと管理者画面は「画面は表示されるが保存できない」
> 状態です（セキュリティルールで守られているため）。設定後に有効になります。

対象プロジェクト: **rank-cf9fe**（既存のランキングと同じプロジェクト）

---

## ステップ1. 管理者ログインを有効にする（Authentication）

1. [Firebase Console](https://console.firebase.google.com/) → プロジェクト **rank-cf9fe** を開く
2. 左メニュー **構築 → Authentication** → 「始める」
3. **Sign-in method** タブ → **メール/パスワード** を選び **有効にする** → 保存

## ステップ2. 管理者アカウントを1つ作る

1. Authentication → **Users** タブ → **ユーザーを追加**
2. あなたの管理用メールアドレスと、推測されにくいパスワードを入力 → 追加
3. 追加されたユーザーの行にある **ユーザーUID**（長い英数字）をコピーしておく
   - これが「管理者の鍵」になります。他人に教えないでください。

## ステップ3. セキュリティルールを公開する

1. 左メニュー **構築 → Firestore Database** → **ルール** タブ
2. このリポジトリの **`firebase/firestore.rules`** の中身を全部コピー
3. ファイル内の `PASTE_ADMIN_UID_HERE` を、ステップ2でコピーした **UID** に置き換える
   （シングルクォート `'` は残したまま、中の文字だけ差し替え）
4. Console のルール欄に貼り付けて **公開**

### 公開後すぐに確認（重要）

- 既存のゲームを開いて、**グローバルランキングがこれまで通り表示・投稿できるか**を確認。
  - このルールは既存の `ranks〇〇` を現状維持する設計ですが、念のため確認してください。
  - もし万一ランキングが動かなくなったら、**すぐに以前のルールに戻して**ください
    （Console のルールは履歴から復元できます）。連絡をもらえれば調整します。

## ステップ4. 動作テスト

1. サイトの **`/admin/`** を開く → ステップ2のメール/パスワードでログイン
   → 申請一覧（最初は空）が表示されれば成功
2. 別タブで **`/teachers/register/`** を開き、テスト申請を送信
3. `/admin/` を再読み込み → 申請が表示される → **承認** を押す
   → 学校コードが発行され、先生への連絡用メール下書き（mailto）が開く

---

## データの保存場所（プライバシー設計）

| コレクション | 内容 | 誰が読めるか |
|---|---|---|
| `school_applications` | 申請（学校名・メール・教科） | **管理者のみ**（第三者は読めない） |
| `schools` | 公開情報（学校名・都道府県・コード） | 全員（メールは含めない） |
| `school_private` | 先生のメールアドレス等 | **管理者のみ** |
| `school_scores` | 学校内ランキング | 学校コードを知る人 |
| `ranks〇〇`（既存） | グローバルランキング | 全員（現状維持） |

- **生徒の個人情報は一切保存しません**（名前・行動ログなし）。
- 先生のメールアドレスは `school_private` と申請データにのみ入り、いずれも
  **管理者UIDを持つあなたしか読めません**。第三者が登録ログやDBを覗くことはできません。

## （任意）承認メールを自動送信したい場合

現状は承認時に「mailto」でメール下書きが開く方式（追加設定ゼロ）です。
自動送信にしたい場合は EmailJS でテンプレートを作り、`admin/index.html` の
`SCHOOL_APPROVE_TEMPLATE` に ID を設定すれば切り替えられます（任意・後回しでOK）。
